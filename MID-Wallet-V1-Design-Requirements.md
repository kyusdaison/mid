# MID Wallet V1 设计需求文档

## 文档状态
- 版本：V1.0 Draft
- 日期：2026-03-17
- 用途：用于统一 MID Wallet 第一版的产品定位、设计边界、页面结构与验收标准

## 来源依据
本文件基于当前项目中的以下材料整理而成：
- `mid-wallet.html`
- `mid-wallet-app/`
- `montserrat-digital-residency.html`
- `MID-Wallet-Upgrade-Roadmap.md`

本文件是设计需求文档，不是技术实现文档。

## 1. 产品定义
MID Wallet V1 不是一个普通的加密钱包，也不是一个超级应用入口。

它的正确定位应当是：

`一个服务于 Montserrat Digital Residency 的主权身份优先钱包。`

也就是说，钱包的第一职责不是交易，不是理财，不是功能堆叠，而是：

1. 持有身份
2. 解锁身份
3. 展示身份
4. 使用身份
5. 在身份之上承载基础资产能力

## 2. V1 核心目标
第一版必须清楚证明以下四件事：

1. 用户拥有一个可信的政府支持数字身份。
2. 这个身份可以在钱包中被安全解锁。
3. 这个身份可以被出示、验证或调用。
4. 钱包可以承载基础资产信息，但不会掩盖“身份优先”的产品定位。

## 3. V1 设计总原则
所有 V1 设计决策都应遵循以下原则：

1. 身份优先，资产第二。
2. 信任优先，炫技第二。
3. 证据优先，口号第二。
4. 少而稳，强于多而散。
5. 政府级可信感，优先于 crypto 圈层感。

## 4. 核心用户
V1 主要服务三类认知场景：

1. 数字居民本人：需要在钱包中查看和使用自己的数字身份。
2. 验证对象或机构：需要看到一个可信、清晰、可操作的验证流程。
3. 外部观察者或合作方：需要快速判断这个产品是否真实、严肃、可落地。

## 5. V1 核心体验主线
第一版的钱包体验应围绕以下主线展开：

1. 用户打开钱包。
2. 用户先看到身份处于受保护状态。
3. 用户通过生物识别或设备验证解锁身份。
4. 用户首先看到数字身份卡。
5. 用户随后看到钱包总览与基础资产。
6. 用户进入验证流程，出示或验证某项凭证。

如果某个页面或功能无法服务这条主线，就不应进入 V1 核心设计。

## 6. V1 范围定义

### V1 必须包含
1. 锁定态入口页
2. 身份解锁流程
3. 数字身份卡
4. 钱包首页
5. 核心操作区
6. 基础资产列表
7. 验证中心
8. 基础历史记录页

### V1 明确不做
1. 社交流
2. AI 代理
3. 复杂 DeFi
4. 群体储蓄
5. 继承规划
6. 碳足迹
7. 迷你应用生态
8. 超级应用式入口墙
9. 过重的金融分析面板

这些内容可以保留在路线图中，但不能影响第一版设计判断。

## 7. 信息架构要求

### 钱包首页推荐结构
首页层级建议固定为：

1. 身份锁定或解锁状态
2. 数字身份卡
3. 钱包总览卡
4. 核心操作按钮
5. 资产列表
6. 最近活动预览（可选）

首页不能变成功能超市，不能把注意力从身份核心上移走。

### 推荐一级导航
V1 一级导航建议围绕以下四个主面展开：

1. Home
2. Verify
3. History
4. Settings

如果再加太多主入口，V1 的主线会被打散。

## 8. 模块设计要求

### 8.1 锁定态入口页
这是用户进入钱包后的第一印象。

设计要求：
- 必须传达“身份受到保护”，而不是“应用加载失败”。
- 必须明确说明进入钱包前需要身份验证。
- 页面只能有一个主要动作，不应分散选择。
- 气质要稳重、克制、有控制感。

用户看到这一页时，应该产生的判断是：

`这个钱包在保护我的身份访问权。`

而不是：

`这个应用还没准备好。`

### 8.2 身份解锁流程
身份解锁不是普通登录动作，而是产品的核心仪式。

设计要求：
- 优先支持生物识别。
- 必须有清晰的“验证中”状态。
- 成功后应自然过渡到身份卡页面。
- 失败状态必须清晰、克制、不惊慌。
- 整个流程应更像“身份证明被激活”，而不是“进入 App”。

### 8.3 数字身份卡
这是 V1 最重要的视觉对象。

卡片必须明确承载以下信息：
- 签发主体
- 身份类型
- 持有人
- 身份编号或 Residency 编号
- 有效期
- 当前验证或注册状态

卡片设计要求：
- 必须像主权凭证，不能像会员卡或普通银行卡。
- 信息清晰度高于装饰效果。
- 动效只能是辅助，不能喧宾夺主。
- 即使截成静态图，也必须具备可信度。
- 字体、层级、留白要体现证件系统感，而不是营销感。

建议：
V1 卡面采用更克制、更机构化的方向作为唯一母版，不继续分裂多套审美路线。

### 8.4 钱包总览卡
钱包总览卡是辅助身份卡的，不应反客为主。

设计要求：
- 清晰展示总资产或核心余额。
- 视觉重要性低于身份卡。
- 风格上必须属于同一套“可信系统”，不能变成消费金融广告牌。
- 不要做成强烈的信用卡隐喻，除非有明确产品目的。

钱包总览的作用是支持“身份钱包”概念，不是把产品重新定义为交易钱包。

### 8.5 核心操作区
V1 只保留最必要的三项动作：

1. Receive
2. Send
3. Verify / Scan

设计要求：
- 名称必须直接、易懂。
- 图标必须简洁、制度化。
- 点击反馈必须明确。
- 不能在这一层塞入探索性功能和未来功能。

### 8.6 资产列表
资产列表的目标是证明钱包具备基础承载能力，而不是做交易终端。

设计要求：
- 仅展示有限数量的核心资产。
- 优先保证清晰、稳定、低噪音。
- 数值格式、涨跌、币种标识必须统一。
- 不需要堆叠过多行情信息。

资产列表要回答的问题是：

`我持有什么？`

而不是：

`我现在该怎么高频交易？`

### 8.7 验证中心
验证中心是 V1 最重要的产品证明页面之一。

设计要求：
- 必须显示可用凭证。
- 至少支持一种主验证方式和一种备用方式。
- 选择性披露必须作为核心能力出现。
- 验证过程必须让非技术用户也能理解。
- 页面应体现“可实际使用”，而不是“概念展示”。

这是 MID Wallet 从“好看原型”走向“真实身份产品”的关键页面。

### 8.8 历史记录页
历史记录在 V1 中可以基础化处理，但不能缺少状态清晰度。

设计要求：
- 展示近期记录或动作日志。
- 尽可能区分身份相关事件与资产相关事件。
- 强调状态：confirmed、pending、locked、failed、expired。

它的作用是增强可追溯性与可信度，而不是增强交易分析能力。

## 9. 视觉语言要求

### 整体气质
钱包整体应当让人感受到：
- 主权感
- 控制感
- 精密感
- 隐私感
- 现代感

不应让人感受到：
- 投机感
- 噪音感
- 过度游戏化
- 为未来感而未来感

### 颜色
- 延续深色中性底和克制的冷蓝强调色。
- 绿色只用于 verified、active 等积极状态。
- 强调色要有系统意义，不能只是装饰。

### 字体
- 要兼具机构严肃性和数字系统精度。
- 标签和字段信息应带有正式、结构化气质。
- 大标题和展示型字体应节制使用。
- 证件字段始终比营销语更重要。

### 布局
- 层级必须非常强。
- 留白要服务可信感与阅读速度。
- 卡片布局应支持用户在数秒内完成识别。

## 10. 动效要求
动效必须服务理解，不能沦为表演。

设计要求：
- 过渡短、稳、低噪音
- 每个动作都有明确的起止状态
- 在安全动作上提供适度仪式感
- 不允许常驻动态削弱阅读信心

动效适合用于：
- 解锁成功
- 验证处理中
- 页面切换
- 轻量环境氛围

动效不应用来掩盖信息结构不清或内容不足。

## 11. 文案要求
V1 文案必须遵循以下规则：

1. 简短
2. 稳重
3. 官方
4. 围绕状态与动作
5. 少形容，多事实

推荐使用的语言类型：
- Verified
- Active
- Issued
- Expires
- Credential
- Registry
- Proof
- Secure

尽量少在产品界面里使用以下表达：
- Revolutionary
- Premium
- Borderless
- Next-generation
- Futuristic

这些词可以用于外部宣传，但不应主导产品 UI。

## 12. 必须覆盖的状态
V1 设计必须完整覆盖以下状态：

1. 身份未注册
2. 身份已注册但未解锁
3. 验证中
4. 验证成功
5. 验证失败
6. 无资产
7. 有资产
8. 网络不稳定或离线
9. 凭证过期
10. 凭证有效
11. 交易待处理
12. 交易受限或锁定

如果这些状态没有被系统化设计，第一版再精美也会显得不完整。

## 13. V1 设计边界
为了保护第一版清晰度，必须避免以下偏差：

1. 把首页做成交易所
2. 把钱包总览做得比身份卡更强势
3. 把动画错当成产品成熟度
4. 在主闭环未成立前增加过多概念页面
5. 用路线图野心干扰第一版首次体验

## 14. V1 验收标准
如果一个首次接触项目的人能在 3 分钟内理解以下四件事，说明 V1 设计方向是对的：

1. 这是一个政府支持的身份钱包。
2. 身份是产品中的第一对象，不是余额。
3. 钱包可以安全地解锁并验证该身份。
4. 钱包具备基础资产能力，但不会盖过身份叙事。

## 15. 最终设计方向结论
MID Wallet V1 的设计方向应当被统一为：

`一个克制的、主权化的、身份优先的钱包产品。`

它必须同时满足两件事：

1. 放在政府数字身份语境里看，足够可信。
2. 放在现代数字产品语境里看，足够好用。

如果某个设计决策让钱包更像投机型交易产品、科幻概念界面、或超级应用外壳，那么它就不适合 V1。
